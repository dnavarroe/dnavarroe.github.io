Recommendation systems

Description

An online book-selling platform aims to enhance user experience by providing personalized recommendations. They recognize the importance of offering relevant suggestions based on individual user tastes to increase customer satisfaction and encourage platform loyalty. They decide to develop a book recommendation system based on authors to achieve this goal.

Problem Description

The main challenge is how to design an effective recommendation system that suggests books based on users' preferred authors. Choosing a specific author is often a strong indicator of an individual's literary interests. Therefore, the system should be able to analyze a user's reading history and recommend books written by similar or related authors.

Available Data

  • Reading History:Information about the books each user has previously read, including authors and genres.

  • Book Catalog:A database of books available on the platform, including detailed information about authors, genres, user ratings, etc.

Objective

The main objective is to develop a recommendation system that suggests books likely to be enjoyed by a user based on their favorite authors. By providing accurate and personalized recommendations, the platform aims to increase user engagement and book sales.

Expected Benefits

  • Improvement in user experience by receiving personalized and relevant recommendations.

  • Increase in user engagement and retention on the platform.

  • Growth in book sales by effectively promoting specific titles.

  • Enhancement of brand perception as a platform that understands and addresses individual user needs.

Surprise

In this notebook we are going to use the surprise library. This is a library that is based on the scikit-learn API and allows you to implement several basic recommendation algorithms.

Let's start by loading a classic dataset in recommender systems: MovieLens (https://movielens.org/). This is a movie recommendation page that opened historical information.

The surprise package does not directly receive a DataFrame object but rather has to parse and read a set of data through two new objects: Reader and Dataset. In Reader we must specify the minimum value and maximum value of the ratings and Dataset allows us to read data from different sources.

Now load SVD and GridSearchCV, both from surprise. Note: GridSearchCV is not in surprise.GridSearchCV, surprise.GridSearch is deprecated.

Generate a parameter grid where different combinations of:

  • epochs: is the number of passes over the dataset that the algorithm will make using gradient descent
  • biased: use bias parameters or not
  • lr_all: learning rate for all parameters
  • reg_all: regularization term for all parameters (lambda)

Use GridSearchCV, SVD, and the dictionary with the parameters to test, and train a model. Note that GridSearchCV needs to be passed an uninstantiated model. Also, set the refit parameter to True and with measures = ["rmse","fcp"]

Print the rmse and fcp, and the best combination of parameters

Save the model with the highest fcp and predict the rating for user id 196 and item id 242

We are going to calculate the term occurrence matrices using sklearn.

Both classes first build the total vocabulary, and then:

  • CountVectorizer returns the absolute frequency of each term for each document.
  • TF-IDF:calculates the frequency of each term per document, and normalizes by the total number of documents where the term appears.
